# GigsForGigs

> A specialized marketplace connecting startups and task owners with skilled gig professionals for outcome-driven micro-gigs.

---

# Problem Statement

Managing the onboarding and profile system of gig professionals for a specialized marketplace connecting startups and clients with skilled talent for short-term, outcome-driven tasks.

## For Clients

* Need specific tasks completed quickly (landing pages, pitch decks, product assets)
* Traditional hiring is too slow and expensive for short-term work
* Large platforms like Upwork are noisy, resume-heavy, and often expensive
* Difficulty managing multiple small tasks efficiently

## For Gig Professionals

* Skilled students and professionals struggle with discoverability
* Limited opportunities to build portfolios with real-world projects
* Lack of structured credibility systems
* Difficulty converting skills into real opportunities

### The Gap

Clients require **fast execution of clearly defined tasks**, while skilled professionals seek **real-world projects to build portfolios and credibility**.

Traditional freelance platforms fail to bridge this gap effectively.

---

# Solution Overview

**GigsForGigs** solves the speed vs quality trade-off by focusing on **task-based micro-gigs with clearly defined deliverables**.

Instead of long-term hiring, the platform enables clients to post **specific outcome-driven tasks**, which gig professionals can apply for and complete efficiently.

---

# Platform Actors

## 1. Gig Professional

Gig professionals are individuals offering specialized skills to complete tasks posted by clients.

### Responsibilities

* Create a professional profile
* List skills and portfolio work
* Browse available task opportunities
* Accept or decline service requests
* Submit deliverables
* Build a portfolio through completed tasks
* Review clients after task completion

---

## 2. Client

A client represents anyone who wants to hire gig professionals for tasks.

Clients can manage tasks directly or add managers to help coordinate hiring and execution.

### Responsibilities

* Post tasks
* Search and hire gig professionals
* Monitor task progress
* Approve deliverables
* Make payments
* Leave reviews

---

## 3. Root Client

The Root Client is the **primary account owner** for a client workspace.

The root client has full control over hiring and management.

### Responsibilities

* Add and manage manager accounts
* Approve hiring decisions
* Oversee all tasks
* Monitor task progress
* Manage payments

---

## 4. Manager

Managers are accounts added by the Root Client to assist in task management.

Managers help coordinate hiring and task execution.

### Responsibilities

* Post tasks
* Search gig professionals
* Monitor task progress
* Manage task workflows

Managers assist with operations but **final approval authority belongs to the Root Client**.

* Monitor task progress

# Key Terminology

| Term                     | Definition                                     |
| ------------------------ | ---------------------------------------------- |
| **Micro-gigs**           | Small tasks with defined outcomes              |
| **Gig Professional**     | Skilled individual completing tasks            |
| **Supply**               | Gig professionals available on the platform    |
| **Demand**               | Clients posting tasks                          |
| **Outcome-driven tasks** | Tasks with clearly defined deliverables        |
| **Task Opportunity**     | Work posted by clients for gig professionals   |
| **Deliverable**          | Completed work submitted by gig professionals  |
| **Testimonial**          | Reviews and ratings left after task completion |
| **Portfolio Building**   | Using completed work to demonstrate expertise  |

# Platform Workflow

The lifecycle of a task on the platform:

Client posts task
↓
Gig professionals browse task opportunities
↓
Client searches and selects gig professional
↓
Root client approves hiring (if required)
↓
Gig professional accepts service request
↓
Task execution
↓
Deliverable submission
↓
Client approval
↓
Payment release
↓
Mutual reviews
